## ADDED Requirements

### Requirement: 系统能够实时计算 OpenCV 模板匹配结果
系统 SHALL 使用 `TM_CCOEFF_NORMED` 算法实时计算模板匹配，并返回匹配位置、置信度、尺寸与耗时。

#### Scenario: 成功匹配并显示结果
- **WHEN** 用户加载模板并点击“匹配”按钮
- **THEN** 系统异步执行模板匹配
- **THEN** 画布绘制匹配结果框并显示置信度与耗时

#### Scenario: 多个匹配结果
- **WHEN** 模板存在多个高于阈值的匹配位置
- **THEN** 系统返回所有满足阈值的匹配结果
- **THEN** 画布按置信度着色显示所有匹配框

### Requirement: 系统能够支持阈值滑块实时调节
系统 SHALL 提供阈值滑块，范围 0.50 到 0.95，用户调节时实时更新匹配结果显示。

#### Scenario: 调节阈值更新匹配结果
- **WHEN** 用户调整阈值滑块
- **THEN** 系统根据当前结果重新过滤或重新显示匹配项
- **THEN** 用户可立即观察匹配变化

### Requirement: 系统能够输出匹配测试报告
系统 SHALL 输出匹配测试报告，包含状态、耗时、坐标、置信度、匹配数量与失败原因。

#### Scenario: 输出成功匹配报告
- **WHEN** 匹配成功
- **THEN** 报告显示成功状态、耗时、位置与置信度
- **THEN** 用户可直接据此调整模板或阈值

#### Scenario: 输出失败匹配报告
- **WHEN** 匹配失败
- **THEN** 报告显示最高置信度和当前阈值
- **THEN** 用户可据此判断是否需要调整模板或 region

### Requirement: 系统能够验证控件选择器有效性
系统 SHALL 验证生成的 UiSelector 对当前 UI 树是否有效，并将结果反馈到树与画布。

#### Scenario: 选择器验证成功
- **WHEN** 用户点击“验证选择器”按钮且存在匹配节点
- **THEN** 系统高亮显示匹配节点
- **THEN** 报告显示匹配数量

#### Scenario: 选择器验证失败
- **WHEN** 选择器在当前 UI 树中无匹配节点
- **THEN** 系统显示验证失败提示
- **THEN** 报告建议检查 resource-id、text 或 content-desc

### Requirement: 系统能够验证 Dump 坐标与截图坐标对齐
系统 SHALL 对比 UI dump bounds 与截图像素坐标，帮助用户确认控件框位置准确性。

#### Scenario: 坐标对齐验证成功
- **WHEN** 用户点击“验证坐标对齐”按钮且偏差在允许范围内
- **THEN** 系统输出对齐结果与误差值
- **THEN** 用户可确认 overlay 位置可信

#### Scenario: 坐标对齐失败
- **WHEN** 控件边界框与截图位置偏差超出阈值
- **THEN** 系统输出警告与详细偏差数据
- **THEN** 用户可进一步检查分辨率、方向或截图来源

### Requirement: 系统能够支持批量模板匹配测试
系统 SHALL 支持加载多个模板图像，批量执行匹配测试并生成汇总报告。

#### Scenario: 批量匹配测试
- **WHEN** 用户加载多个模板并点击“批量匹配”
- **THEN** 系统依次执行每个模板的匹配测试
- **THEN** 汇总报告显示成功数、失败数与总耗时
